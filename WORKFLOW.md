# Workflow

## 목적

이 프로젝트는 반복 지시 없이 작업이 이어지도록 고정된 전달 루프를 사용한다.

## 표준 순서

1. 계획 수립
   - 대상 작업을 정한다.
   - 완료 기준을 정한다.
   - 검증 방법을 정한다.
2. 계획 검토
   - 리스크, 회귀 가능성, 의존성 누락, 범위 오류를 확인한다.
3. 작업 수행
   - 가장 작은 완료 가능한 단위를 구현한다.
4. 검증
   - 정적 검증, 타깃 테스트, 필요 시 더 넓은 회귀 테스트를 수행한다.
5. 문서 반영
   - `README.md`, `SPEC.md`, `TODO.md`, `NEXT_SESSION_PLAN.md`를 현재 상태에 맞춘다.
6. 커밋
   - 완료된 단계마다 집중된 커밋 하나를 남긴다.
7. 다음 항목 진행
   - 가장 우선순위 높은 미완료 항목으로 자동 이동한다.

## 현재 우선순위 기준

1. `TODO.md`
2. `NEXT_SESSION_PLAN.md`
3. `SPEC.md`
4. `README.md`

## 현재 MVP 범위

- 웹 UI 기반 로컬 RAG
- 사용자 경로: `intro -> app`
- 관리자 경로: `intro -> admin`
- 핵심 기능: 질의, 문서 조회, 업로드 요청, 관리자 승인

## 기본 보류 항목

- CLI 제품화
- 데스크톱 래핑
- GraphRAG 통합(잠정 중단)

## 멈추고 물어봐야 할 때

- 파괴적 작업이 필요한 경우
- 범위나 아키텍처 결정이 불명확한 경우
- 외부 자격증명 또는 유료 서비스가 필요한 경우
- 충분한 검증 없이 결과를 확정할 수 없는 경우

## 완료 정의

아래가 모두 끝나야 작업 완료로 본다.

- 구현 완료
- 관련 검증 완료
- 문서 갱신 완료
- 커밋 완료
