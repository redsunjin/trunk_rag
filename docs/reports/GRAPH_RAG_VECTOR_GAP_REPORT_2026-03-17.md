# GraphRAG Vector Gap Report (2026-03-17)

## 목적
- 현재 Vector RAG 경로가 어디까지는 충분하고, 어디서부터 관계형 보강이 필요한지 정리한다.
- GraphRAG를 전체 기본 경로가 아니라 특정 질문군에만 좁혀 적용하기 위한 근거를 남긴다.

## 확인된 사실

### 1) 교차 국가 비교는 `collection=all`로 충분하지 않았다
- 근거:
  - `docs/reports/QUERY_QUALITY_ROUTE_REPORT_2026-03-15.md`
- 확인된 사례:
  - 질의: `프랑스의 에콜 폴리테크니크와 독일의 훔볼트 대학은 과학 인재 양성에서 어떤 역할을 했는지 비교해줘`
  - `collection=all` 고정 시 두 모드 모두 `제공된 문서에서 확인되지 않습니다.`
  - 같은 질문을 자동 다중 라우팅(`fr,ge`)으로 처리하면 비교 응답 가능
- 의미:
  - 현재 Vector RAG도 라우팅이 맞으면 해결 가능한 질문이 있다.
  - 즉, 모든 비교 질문이 곧 GraphRAG 대상은 아니다.

### 2) 다중 컬렉션은 커버리지를 주지만 지연 비용이 크다
- 근거:
  - `docs/reports/MULTI_COLLECTION_POC_REPORT_2026-02-26.md`
- 확인된 수치:
  - `dual_fr_ge` p95는 `single_fr` 대비 약 `+82.7%`
  - 평균 문서 수와 source 수는 약 `2배`
- 의미:
  - 관계형 질문을 단순히 "더 많은 컬렉션 검색"으로 풀면 지연이 빠르게 올라간다.
  - 3개 이상 국가/기관을 엮는 질문은 현재 방식으로 확장할수록 비용이 더 커질 가능성이 높다.

## 구조상 한계

아래는 현재 아키텍처와 질문셋을 바탕으로 한 추론이다.

### 3) 현재 라우팅 상한은 2개 컬렉션이라 3개 이상 연결 질문을 직접 다루기 어렵다
- 근거:
  - 현재 기본 정책은 자동 다중 라우팅 최대 2개 컬렉션
  - 질문셋의 `graph-candidate` 중 다수는 `fr,ge,uk` 또는 `fr,ge,uk,it`를 동시에 요구
- 영향 질문:
  - `GQ-09`, `GQ-11`, `GQ-14`, `GQ-15`, `GQ-18`
- 해석:
  - 이 질문군은 검색 커버리지부터 현재 기본 경로 상한을 넘는다.

### 4) 현재 검색은 관계 사슬 복원보다 유사 청크 검색에 강하다
- 근거:
  - 현재 구조는 chunk 기반 벡터 검색 + 상위 문맥 조합
  - 명시적인 entity/relation 저장 계층이 없다
- 영향 질문:
  - 사건 -> 인물 -> 제도 -> 산업화처럼 단계가 긴 `GQ-12`, `GQ-14`, `GQ-16`, `GQ-18`
- 해석:
  - 질문이 길어질수록 "관련 문장을 모으는 것"과 "관계를 복원하는 것" 사이 간극이 커진다.

## 현재 Vector RAG로 계속 가져갈 질문군
- `ops-baseline`
- 1개 국가 설명
- 2개 국가 비교
- 이미 문서 안에 직접 요약된 institution/person 중심 질문

대표 예:
- `GQ-01` ~ `GQ-08`

## GraphRAG sidecar 후보 질문군
- 3개 이상 국가/기관/인물 연결
- 다중 문서에서 관계 사슬을 복원해야 하는 질문
- 동일 인물/기관이 다른 국가 제도에 남긴 전이 효과를 설명하는 질문

대표 예:
- `GQ-09` ~ `GQ-18`

## 개선 필요성 결론
1. GraphRAG는 현재 `/query` 전체를 대체할 필요가 없다.
2. 현재 Vector RAG는 `ops-baseline` 질문에는 계속 기본 경로로 유지하는 편이 맞다.
3. GraphRAG sidecar는 `graph-candidate` 질문군만 좁혀서 검토해야 한다.
4. 다음 단계는 질문셋 기준으로 실패/성공 응답 예시를 붙이고, 그 다음에만 사이드카 계약을 정의하는 순서가 맞다.
